HTML ka matlab kya hai?

HTML ek markup language hai jo web pages ki structure banati hai. HTML me <tags> hote hain jo browser ko batate hain ki content ka role kya hai (heading, paragraph, image, link, form, etc.).

  • HyperText = Links ke through pages ka connection.
  • Markup = Tags ke zariye meaning/semantics dena.
  • Language = Rules/syntax ka set.

Basic Structure (Boilerplate)

Har HTML document ki base structure kuch is tarah hoti hai:

Code: Minimal HTML5 Document 
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>Page Title</title>
  </head>
  <body>
    <h1>Hello, World!</h1>
  </body>
</html>
<!DOCTYPE html>

Browser ko batata hai ki ye document HTML5 standard follow karta hai.

<html lang="en">

Root element + language declare karna accessibility/SEO ke liye accha.

<head>

Metadata, CSS/JS links, title, icons yahan hote hain.

<body>

Page ka visible content (headings, paragraphs, images, etc.).

Text & Headings

Headings hierarchical hote hain: h1 (main) se h6 (sub‑subsection) tak. Text ke liye p, emphasis ke liye em/strong, aur inline semantics ke liye span, mark, small, sup/sub.

Code: Headings & Inline Text 
<h1>Page Title</h1>
<h2>Section Title</h2>
<p>This is a <strong>paragraph</strong> with <em>emphasis</em>, 
<mark>highlight</mark>, and H<sub>2</sub>O.</p>
<h1>…</h1> <p>…</p> <em> <strong> <span> <mark> <small> <abbr title> <cite> <code> <kbd> <sub>/<sup>

Links & Images

Links ke liye <a> (with href), images ke liye <img> (with src + alt).

Code: Links & Images 
<a href="https://developer.mozilla.org/" target="_blank" rel="noopener">MDN Docs</a>
<img src="https://picsum.photos/400/200" alt="Random placeholder" width="400" height="200">

target="_blank" naya tab kholta hai; rel="noopener" security best‑practice.

Lists

Tables

Table structure: table → caption, thead, tbody, tfoot, rows tr, headers th, cells td.

Code: Accessible Table 
<table>
  <caption>Monthly Sales</caption>
  <thead>
    <tr><th scope="col">Month</th><th scope="col">Amount</th></tr>
  </thead>
  <tbody>
    <tr><th scope="row">Jan</th><td>₹12,000</td></tr>
    <tr><th scope="row">Feb</th><td>₹15,500</td></tr>
  </tbody>
  <tfoot>
    <tr><th scope="row">Total</th><td>₹27,500</td></tr>
  </tfoot>
</table>

Forms

Forms data input ke liye: form, label, input, textarea, select, button, with attributes like required, placeholder, name, type.

Code: Form with Inputs 
<form action="#" method="post">
  <label for="name">Name</label>
  <input id="name" name="name" type="text" placeholder="Your name" required>

  <label for="email">Email</label>
  <input id="email" name="email" type="email" placeholder="you@example.com" required>

  <label for="topic">Topic</label>
  <select id="topic" name="topic">
    <option>General</option>
    <option>Feedback</option>
  </select>

  <label for="msg">Message</label>
  <textarea id="msg" name="message" rows="3"></textarea>

  <fieldset>
    <legend>Preferences</legend>
    <label><input type="checkbox" name="sub" checked> Subscribe</label>
    <label><input type="radio" name="mode" value="light" checked> Light</label>
    <label><input type="radio" name="mode" value="dark"> Dark</label>
  </fieldset>

  <button type="submit">Send</button>
</form>

Common input types: text, email, password, number, date, file, color, range, checkbox, radio, submit.

Semantic Tags

Semantic tags content ka meaning clear karte hain — SEO & accessibility friendly.

<header>

Top section: logo, nav.

<nav>

Navigation links ka group.

<main>

Page ka primary content (unique).

<section>

Topic-based grouping with heading.

<article>

Self‑contained content (blog/news).

<aside>

Side info: ads, related links.

<footer>

Page bottom: copyright, links.

<figure><figcaption>

Media + caption.

<time>

Machine‑readable date/time.

Code: Semantic Layout Snippet 
<header>Site Header</header>
<nav>...links...</nav>
<main>
  <article>
    <h2>Blog Post</h2>
    <p>Content...</p>
  </article>
  <aside>Related</aside>
</main>
<footer>© 2025</footer>

Audio & Video

Native media tags with controls, sources, captions.

Code: Audio & Video 
<audio controls>
  <source src="song.mp3" type="audio/mpeg">
  Your browser does not support audio.
</audio>

<video controls width="420" poster="poster.jpg">
  <source src="clip.mp4" type="video/mp4">
  Sorry, your browser cannot play this video.
</video>

Head/Metadata

Head me SEO, icons, social preview, fonts, CSS/JS links define hote hain.

Code: Common <head> tags 
<meta name="description" content="Page summary">
<meta name="theme-color" content="#0ea5e9">
<link rel="icon" href="/favicon.ico" sizes="any">
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="stylesheet" href="styles.css">
<script defer src="app.js"></script>
<meta property="og:title" content="Your Title">
<meta property="og:image" content="https://example.com/cover.png">
<meta name="twitter:card" content="summary_large_image">

Global Attributes (har tag ke sath kaam aate)

  • id – unique identifier
  • class – CSS/JS grouping
  • style – inline CSS
  • title – tooltip text
  • hidden – element hide
  • tabindex – keyboard focus order
  • aria-* – accessibility annotations

HTML Entities

Special characters likhne ke liye entities use hote: &lt; (<), &gt; (>), &amp; (&), &copy; (©), &nbsp; (non‑breaking space).

Quick Tips

  • Har page me lang attribute zaroor set karo.
  • Images me alt likho — accessibility & SEO ke liye.
  • Forms me label ko inputs se associate karo (via for).
  • Heading order logical rakho: h1 → h2 → h3.
  • Layout ke liye CSS, meaning ke liye HTML.

Practice Ideas

  2. Accessible form create karo with validation attributes.
  3. Responsive table design try karo (stack on small screens).
